# Interface: ElementAudioSource

Defined in: [packages/timeline/src/audio-source.ts:14](https://github.com/mattppal/mcut/blob/main/packages/timeline/src/audio-source.ts#L14)

Normalized answer to "what source audio does this timeline element play?"
UI and agent surfaces can use this before passing bytes to a transcription
provider, waveform analyzer, or any other media-only primitive.

#### src

```ts
src: string;
```

Object URL, data URL, or remote URL. Projects stay JSON-serializable —
but treat this as a RUNTIME BINDING, not identity: object URLs are dead
after a reload, so persistence layers re-resolve `src` on load (e.g. from
OPFS via `hash`) and relink UIs match on `hash`/`name`.
