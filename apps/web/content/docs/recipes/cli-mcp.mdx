---
title: Batch edit with CLI or MCP
description: Use the same command and operator model from tools.
---

The CLI is best for file-based workflows:

```sh
bunx @mcut/cli summarize project.mcut.json
bunx @mcut/cli commands --name addElement
bunx @mcut/cli apply project.mcut.json commands.json --dry-run
```

The MCP server exposes project state to agents:

```json
{
  "mcpServers": {
    "mcut": {
      "command": "bunx",
      "args": ["@mcut/mcp-server", "project.mcut.json"]
    }
  }
}
```

For live browser sessions, use the bridge:

```sh
bunx -p @mcut/mcp-server mcut-mcp-live --editor-url http://localhost:3000/editor
```

Agents should read `get_summary` before editing and prefer operator tools when
the action depends on selection or playhead state.
