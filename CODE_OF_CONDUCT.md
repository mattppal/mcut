# Code of Conduct

mcut is an open-source project for people building video editing tools. The
project works best when technical disagreement stays specific, evidence-based,
and respectful.

## Expected behavior

- Be direct about bugs, risks, and tradeoffs without making it personal.
- Assume good intent, but accept maintainer decisions once they are made.
- Keep issues and pull requests focused on actionable project work.
- Credit prior art without copying license-incompatible code or docs.

## Unacceptable behavior

- Harassment, threats, discriminatory language, or personal attacks.
- Publishing private information without consent.
- Repeatedly derailing issues, reviews, or discussions after being asked to stop.
- Submitting copied code, identifiers, or documentation from incompatible
  sources.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests;
temporarily limit participation; or ban contributors from the project. Report
conduct concerns privately to the repository owner or through GitHub's reporting
tools.
