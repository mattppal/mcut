# Governance

mcut is maintained by the project owner and invited maintainers.

## Roles

- Owner: final responsibility for project direction, releases, repository
  settings, npm access, and security response.
- Maintainer: trusted contributor with review and merge authority for assigned
  areas.
- Contributor: anyone proposing issues, documentation, tests, or code through
  pull requests.

## Decision making

Most decisions happen in issues and pull requests. Maintainers should prefer
small, reversible changes and record larger API or package-boundary decisions in
the relevant issue or pull request.

The owner has final say when a decision affects licensing, package ownership,
security posture, public API direction, release timing, or project scope.

## Maintainer access

Maintainer access is granted intentionally and can be removed if someone is no
longer active, no longer follows the clean-room policy, or no longer needs write
access. Contributors do not need write access to participate.

## Clean-room requirement

All maintainers and contributors must follow the clean-room policy in
CONTRIBUTING.md. This is a project-level requirement, not only a pull request
checklist item.
